Quick Guide to daal4py and daal4r

Getting Started

This is a technical preview, not a product. Intel might decide to discontinue this project at any time.

This is an open-source project released under Apache-2.0 license. The sources are available through https://github.com/01org/daal. Nevertheless, we recommend using binaries provided on anaconda.org (see below).

1.1    Python

Supporting Python 2.7 and 3.6 on Linux. It is recommended to use Intel Distribution for Python.

conda create n d4p c intel c intel/label/test daal4py mpi4py
source activate d4p

1.2    R

Supporting R >= 3.3.2 on Windows and Linux.

You can either install pre-built binary packages or build from source.

2.       Install

b.      From command line: R CMD INSTALL <package>

c.       Or from within R: install.packages("<package>)

2      Quick intro

All algorithms work the same way:

1.       Instantiate and parametrize

2.       run/compute on input data

The below tables list the accepted arguments. Parameters passed to the object instantiation start with p_ or t. Input data to the compute methods start with i_. Those with no default (None) are required arguments. All other arguments with defaults are optional and can be provided as keyword arguments (like p_C=77).

Each algorithm returns a dictionary as its result. See the respective DAAL documentation for details.

For algorithms with training and prediction, simply extract the respective entry from the result dict returned by the training and pass it in as the input argument (see the DAAL documentation for details).

Note that all input and output objects are native types, e.g. standard types (integer, Python dict, R named list, ) or numpy arrays in Python or data.frames in R (daal4r also accepts matrix besides data.frame).

3      Single process/batch processing

Input data can be provided by csv-file-name or table (see above for accepted types). See below for other arguments.

3.1    Python examples

import daal4py as d

from numpy import loadtxt

from time import gmtime, strftime

 

def pca():

# let's pass in a file directly, not a table/array

return d.pca().compute("../data/batch/pca_normalized.csv")

 

def svd():

# let's pass in a table/array, not a file

dfin = loadtxt("../data/batch/svd.csv", delimiter=',')

return d.svd().compute(dfin)

 

def mvod():

dfin = loadtxt("../data/batch/outlierdetection.csv", delimiter=',')

return d.multivariate_outlier_detection().compute(dfin)

 

def uvod():

dfin = loadtxt("../data/batch/outlierdetection.csv", delimiter=',')

return d.univariate_outlier_detection().compute(dfin)

 

def kmeans():

dfin = loadtxt("../data/batch/kmeans_dense.csv", delimiter=',')

centroids = d.kmeans_init(10, t_method="plusPlusDense")

return d.kmeans(10).compute(dfin, centroids.compute(dfin))

 

def naive_bayes():

data = loadtxt(".csv", delimiter=',', usecols=range(20))

gt = loadtxt(".csv", delimiter=',', usecols=range(20,21))

gt.shape = (gt.size,1)

res = d.multinomial_naive_bayes_training(20).compute(data, gt)

tdata = loadtxt(".csv", delimiter=',', usecols=range(20))

return d.multinomial_naive_bayes_prediction(20).compute(tdata, res['model'])

 

def svm():

data = loadtxt(".csv", delimiter=',', usecols=range(20))

gt = loadtxt(".csv", delimiter=',', usecols=range(20,21))

gt.shape = (gt.size,1)

res = d.svm_training(p_cacheSize=600000000).compute(data, gt)

tdata = loadtxt(".csv", delimiter=',', usecols=range(20))

return d.svm_prediction().compute(tdata, res['model'])

 

def svm_rbf():

data = loadtxt("..csv", delimiter=',', usecols=range(20))

gt = loadtxt("..csv", delimiter=',', usecols=range(20,21))

gt.shape = (gt.size,1)

res = d.svm_training(p_cacheSize=600000000, p_kernel=d.rbf()).compute(data, gt)

tdata = loadtxt(".csv", delimiter=',', usecols=range(20))

return d.svm_prediction().compute(tdata, res['model'])

 

def svm_multi():

data = loadtxt("...csv", delimiter=',', usecols=range(20))

gt = loadtxt("...csv", delimiter=',', usecols=range(20,21))

gt.shape = (gt.size,1)

predictor = d.svm_prediction()

trainer = d.svm_training(p_cacheSize=100000000)

res = d.multi_class_classifier_training(p_nClasses=5,
p_prediction=predictor,
p_training=trainer).compute(data, gt)

tdata = loadtxt("...csv", delimiter=',', usecols=range(20))

return d.multi_class_classifier_prediction(p_nClasses=5,
p_prediction=predictor,
p_training=trainer).compute(tdata,
res['model'])

 

def linreg():

data = loadtxt(".csv", delimiter=',', usecols=range(10))

gt = loadtxt(".csv", delimiter=',', usecols=range(10,11))

gt.shape = (gt.size,1)

return d.linear_regression_prediction().compute(tdata, res['model'])

4      Multiprocessing/Distributed processing

Some algorithms can work on several processes (distributed). Currently the following algorithms support distribution:

         pca

         svd

         linear_regression_training

         multinomial_naive_bayes_training

         kmeans and kmeans_init

The recommended distribution mechanism is using Intel MPI library. Sockets are also possible.

Note: before exiting the program a call to daalfini() is required.

4.1    Master-Slave

The Master-Slave mode lets you program as if you had a single process. Distribution is done under the hood. To enable this mode your python program should begin with

4.1.1    Python

import daal4py as d4p
d4p.daalinit()

4.1.2    Examples

import daal4py as d
from numpy import loadtxt
from time import gmtime, strftime
 
# calling init without arguments (-> spmd=False) configures for master-slave processing
d.daalinit()
 
########################################################################################
############## kmeans
# Here we pass in all partitions as tables
 
def kmeans():
 files = ["../../samples/cpp/mpi/data/distributed/kmeans_dense.csv"] * 4
 tables = [loadtxt(f, delimiter=',') for f in files]
 
 # let's use the plusplus method
 centroids = d.kmeans_init(20, t_method="plusPlusDense", distributed=True)
 
 # we could use our centroids computed otherwise for kmeans
 return d.kmeans(20, distributed=True).compute(tables, centroids.compute(tables))
 

############## PCA

# Here we pass in all partitions as numpy-arrays
 
def pca():
 files=["../../samples/cpp/mpi/data/distributed/pca_normalized_1.csv",
 "../../samples/cpp/mpi/data/distributed/pca_normalized_2.csv",
 "../../samples/cpp/mpi/data/distributed/pca_normalized_3.csv",
 "../../samples/cpp/mpi/data/distributed/pca_normalized_4.csv"]
 
 return d.pca(distributed=True).compute([loadtxt(f, delimiter=',') for f in files])
 
############## Naive Bayes
# Here we simply pass in the file-names, but we have several input files!
 
def naive_bayes():
 dfiles = ["../../samples/cpp/mpi/data/distributed/naivebayes_train_dense.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_dense.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_dense.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_dense.csv"]
 lfiles = ["../../samples/cpp/mpi/data/distributed/naivebayes_train_labels.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_labels.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_labels.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_labels.csv"]
 tdfile = "../../samples/cpp/mpi/data/distributed/naivebayes_test_dense.csv"
 tlfile = "../../samples/cpp/mpi/data/distributed/naivebayes_test_labels.csv"
 
 res = d.multinomial_naive_bayes_training(20,

 distributed=True).compute(dfiles, lfiles)
 
 # prediction does not distributed the computation
 data = loadtxt(tdfile, delimiter=',')
 return d.multinomial_naive_bayes_prediction(20).compute(data, res['model'])
 
#### run all
for func, name in [(kmeans, 'kmeans'), (pca, 'pca'), (naive_bayes, 'naive bayes')]:
 try:
 print('####################### ' + name)
 assert '__daalptr__' in func()
 print(strftime("%H:%M:%S", gmtime()) + '\tPASSED')
 except BaseException as ex:
 print(str(ex))
 print(strftime("%H:%M:%S", gmtime()) + '\tFAILED')
 
# we need a fini to prevent processes to terminate prematurely
d.daalfini()

4.2    SPMD (Single Program Multiple Data)

You can also program in SPMD-style, e.g. like the usual MPI style. You do not need mpi4py/Rmpi for daal4py/daal4r, but you can use it if required. Any of the above algorithms return a usable result object/dict only on the root process (0). Note: daal4py provides my_procid and num_procs for this. This mode works like as follows:

1.       Your program begins with daalinit(spmd=True)

2.       Provide one array or csv-file per process. This represents the partitioning of your data.

3.       When calling an algorithm add the argument distributed=True.

4.2.1    Python examples

import daal4py as d
from numpy import loadtxt
from time import gmtime, strftime
 
# calling init with spmd=True configures for SPMD processing
d.daalinit(spmd=True)
 
############## kmeans
def kmeans():
 myfile = "../../samples/cpp/mpi/data/distributed/kmeans_dense.csv"
 mytable = loadtxt(myfile, delimiter=',')
 
 # Let's create the init algorithm only and delay execution
 centroids = d.kmeans_init(20, t_method="plusPlusDense", distributed=True)
 
 # we could use our centroids computed otherwise for kmeans
 return d.kmeans(20, distributed=True).compute(mytable, centroids.compute(mytable))
 
############## PCA
def pca():
 files=["../../samples/cpp/mpi/data/distributed/pca_normalized_1.csv",
 "../../samples/cpp/mpi/data/distributed/pca_normalized_2.csv",
 "../../samples/cpp/mpi/data/distributed/pca_normalized_3.csv",
 "../../samples/cpp/mpi/data/distributed/pca_normalized_4.csv"]
 myfile = files[d.my_procid()]
 
 # Here we pass in a numpy-array
 return d.pca(distributed=True).compute(myfile)
 

############## Naive Bayes

def naive_bayes():
 dfiles = ["../../samples/cpp/mpi/data/distributed/naivebayes_train_dense.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_dense.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_dense.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_dense.csv"]
 lfiles = ["../../samples/cpp/mpi/data/distributed/naivebayes_train_labels.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_labels.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_labels.csv",
 "../../samples/cpp/mpi/data/distributed/naivebayes_train_labels.csv"]
 tdfile = "../../samples/cpp/mpi/data/distributed/naivebayes_test_dense.csv"
 tlfile = "../../samples/cpp/mpi/data/distributed/naivebayes_test_labels.csv"
 mydfile = dfiles[d.my_procid()]
 mylfile = lfiles[d.my_procid()]
 
 # Here simply pass in the "local" file-name(s)
 res = d.multinomial_naive_bayes_training(20,

 distributed=True).compute(mydfile, mylfile)
 if d.my_procid() == 0:
 data = loadtxt(tdfile, delimiter=',')
 return d.multinomial_naive_bayes_prediction(20).compute(data, res['model'])
 return None
 
#### run all
for func, name in [(kmeans, 'kmeans'), (pca, 'pca'), (naive_bayes, 'naive bayes')]:
 try:
 if d.my_procid() == 0:
 print('####################### ' + name)
 res = func()
 if d.my_procid() == 0:
 assert '__daalptr__' in res
 print(strftime("%H:%M:%S", gmtime()) + '\tPASSED')
 except BaseException as ex:
 print(str(ex))
 print(strftime("%H:%M:%S", gmtime()) + '\tFAILED')
 
# we need a fini to prevent processes to terminate prematurely
d.daalfini()

4.3    Starting processes for distributed processing

Currently we support the MPI used by Intel Concurrent Collections (Intel CnC), which is usually Intel MPI Library. Just use mpirun as usual and set the communication-protocol to MPI like this

4.3.1    Python

mpirun genv DIST_CNC=MPI n <num-procs> python <your-program>

4.3.2    R

mpirun genv DIST_CNC=MPI n <num-procs> Rscript <your-program>

or

mpirun genv DIST_CNC=MPI n <num-procs> R --slave --no-restore --file=<your-program>

You can also use tcp/sockets instead of MPI. See https://icnc.github.io/api/distcnc.html#dc_sockets.

5      Accepted Arguments

Please refer to the DAAL documentation to learn about the meanings of the arguments. (ignore leading i_, t_ and p_ to find the respective argument in the DAAL docs).

         Parameters passed to the object instantiation start with p_ or t.

         Input data to the compute methods start with i_.

         Arguments with type Table accept a numpy array in Python and a matrix or data.frame in R.

         Arguments with type TableOrFList accept a file or a table. In distributed mode they accept a file, a table, a list of files or a list of tables.

6      Prerequisites for building from source

6.1    Linux

         A C++ compiler with C++11 support

         Intel Threading Building Blocks (Intel TBB) (https://www.threadingbuildingblocks.org/)

         Intel Concurrent Collections (Intel CnC) version 1.2.00 or later (https://github.com/icnc/icnc)

         Intel Data Analytics Acceleration Library (Intel DAAL) version 2018 (https://github.com/01org/daal)

         To use Intel compiler, set CXX=icpc

6.2    Windows

         Microsoft Visual Studio 2017

         Rtools (https://cran.r-project.org/bin/windows/Rtools/)

         Intel Threading Building Blocks (Intel TBB) (https://www.threadingbuildingblocks.org/)

         Intel Concurrent Collections (Intel CnC) version 1.2.00 or later (https://github.com/icnc/icnc)

         Intel Data Analytics Acceleration Library (Intel DAAL) version 2018 (https://github.com/01org/daal)
